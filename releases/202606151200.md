# Mage Pro Release Notes

**Release ID:** 202606151200  
**Image Tag:** 202606151200  
**Repository:** mage-ai/mage-pro  
**Date:** 2026-06-15

---

## Highlights

- **Multiplayer pipeline editing:** Real-time presence and block-level locking in the pipeline editor prevent lost work and editing collisions.
- **AI "Fix with AI" now everywhere:** AI-powered fix actions for failed pipeline runs, error logs, and directly inside pipeline blocks speed up troubleshooting.
- **Flexible pipeline creation:** The New Pipeline flow is streamlined with modals for creation from scratch, templates, or uploads.
- **Enhanced Data Integrations:** New sources, more flexible auth, and improved sample data for Salesforce Marketing Cloud, Google Business Profile, Google Ads, Microsoft Ads, and BigQuery.
- **Global and workspace navigation:** Restored and expanded Data and Agent product dashboards; improved Settings layout and new workspace landing-page preferences.
- **Trace Audit dashboard:** Admins can now audit API events and changes across the workspace with filtering, search, and export.
- **EKS-powered cloud development:** AWS EKS agent dev slots are now required for visual QA, testing, and feature verification.
- **User and secret management:** Multiplayer-aware user profile editing/logout, Secrets management UI improvements, and Doppler secret interpolation with live TTL cache.
- **Process reliability:** Improved DB safety for triggers, EKS slot stability, and BigQuery destination reliability.

---

## New capabilities

- **Collaborative pipeline editing:**  
  - See who is currently viewing or editing a pipeline.
  - Locked block-level editing and presence indicators prevent conflicting edits.
  - Conflict dialogs surface before autosave or save would overwrite teammates.

- **"Fix with AI" accelerators:**  
  - In pipeline runs and logs: instantly seed the AI assistant with failure details to propose code or config fixes.
  - On pipeline blocks: detect live errors, and trigger AI-driven block fixes without leaving the editor.

- **Trace Audit dashboard (Admins):**  
  - View, search, and filter workspace API event history.
  - Export and review full event context and security origins.
  - Detail panel and layout improvements for clear audit tracing.

- **Workspace landing page preference:**  
  - Users can choose which page opens first when entering a workspace (Pipelines, Monitoring, Runs, or Triggers).
  - Setting is per-browser and can be saved from Preferences.

- **Data and Agents dashboards (preview):**  
  - New navigation entries for Data and Agents in the dashboard sidebar.
  - List and detail views for sources, agents, and product context.

- **Secret management improvements:**  
  - Doppler-backed secret interpolation makes dynamic config easy, with live TTL cache and error diagnostics.
  - UX refinements: resizable Secret panel, better empty-state, and improved table usability.

---

## Improvements

- **Pipeline and run management:**
  - Consolidated Actions dropdown menus for all pipeline run rows, with clear Retry, Cancel, Delete, Edit, View Logs, and Fix with AI flows.
  - "Cancel all running" lets you bulk-cancel in-progress jobs at the project level.
  - Improved run status interactivity and delete confirmation dialogs for smoother operation.

- **Pipeline creation and templates:**
  - Multi-path New Pipeline modal for rapid creation from scratch, by template, or by upload.
  - Browse grouped templates, see images/previews, and prefill pipeline details.

- **Log and error handling:**
  - "Fix with AI" appears prominently for recent errors and failed logs.
  - Logs UI now adjusts height to accommodate AI banners and subheaders.
  - Improved error parsing and highlighting for triple-quoted strings in AI/code output.

- **Integrations, data sources, and sample data:**
  - **Salesforce Marketing Cloud:** Now supports configuring any Data Extension as a stream, with auto-discovery and dynamic schema.
  - **Google Business Profile:** Expanded with more streams (search keywords, reviews, Q&A, posts, media, attributes, categories).
  - **Google Ads:** Service account authentication now supported; improved field compatibility and sample data logic.
  - **Microsoft Ads:** Row normalization and schema fixes for compatibility with BigQuery and analytics endpoints.
  - **BigQuery:** Hyphenated table/dataset names now handled correctly in schema/sample data. Pipeline destination queries now respect a settable timeout.
  - **Sample data and connection testing** show clearer error detail for all supported integrations.

- **User and permission management:**
  - Improved Google sign-in security—no default roles assigned to new users unless configured.
  - Management UIs now leverage avatars, tabs, and consistent styling.

- **Jupyter kernel isolation (beta):**
  - With multi-user kernels enabled, each authenticated user runs code in their own sandboxed Jupyter session.
  - Prevents cross-user code/output leakage in multi-tenant environments.

- **Backend and reliability:**
  - Trigger pipeline schedule lookups are now DB-safe and handle transaction rollbacks automatically.
  - K8s executor CPU usage drastically reduced for prolonged workloads.
  - Memory usage detection improved for containerized/cloud environments.
  - Compatibility with Python 3.11 and 3.12, pandas 2.x, updated dbt 1.10.x, and modern dependencies.

- **Global navigation and UI:**
  - Restored Data Products in the dashboard nav (now under `global-data`).
  - Settings gets a first-class avatar selector and security shortcuts.
  - Shared modals, tabs, and panels (Mana) have refined spacing, borders, and accessibility.
  - Storybook displays all Mana icon versions without cap.

- **Secret and credential safety:**
  - User-scoped Git personal access tokens stay isolated and not promoted to project-wide preferences.
  - Doppler secrets support interpolation with short-lived cache, aiding dynamic environments and secret rotation.

- **Test and process QA:**
  - Automated "agent dev" slots are EKS-only and audited for capacity, health, and scheduled cleanup.
  - Playwright and frontend type checking are routed through EKS agent pods for production-like validation.
  - Animated visual QA now requires video evidence, elevating QA standards for UI motion/animation.

- **Release publishing and changelog:**
  - Release workflows now produce product-facing changelogs, tied to image tags and surfaced on Home.
  - Only user-facing and customer-impacting changes are included in public release notes.

---

## Reliability and fixes

- **Pipeline loading errors:** Handling missing `uuid` in block metadata avoids KeyErrors on graph pages.
- **BigQuery destination queries:** Stuck pipeline runs prevented by a new API-level timeout for queries.
- **K8s executor:** Patched excessive CPU use during API deserialization.
- **Google sign-in:** No role leakages—new users have no project access unless explicitly allowed.
- **Doppler secrets:** Clear error states for missing or expired secrets; disables cache with a config override.
- **Trace Audit layout:** Empty-state, table borders, and resizable right panels now work as expected (no double-borders).
- **Pipeline schedule triggers:** API-level protection against DB transaction rollbacks.
- **Frontend triggers page:** Malformed cron intervals no longer cause runtime errors on status-sort.
