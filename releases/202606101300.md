# Mage Pro Release Notes

**Release ID:** `202606101300`  
**Release date:** 2026-06-10

---

## Highlights

- **Pipeline Editor is now multiplayer.** Real-time presence, block-level editing locks, and autosave collision prevention dramatically improve collaboration and data integrity when working with teammates.
- **Home dashboard surfaces release notes.** Users now see a well-structured "What's new" panel with the latest Mage Pro product updates matching their installed version.
- **Release notes only include user-facing features.** Internal engineering and maintenance work are excluded from public change logs, keeping release communications product-focused.
- **New cloud QA/dev workflows are EKS-only.** All visual QA, browser proof, and agent work now use AWS EKS agent dev slots as the only supported path.
- **Data integration: New sources and improved compatibility.** Major data integration enhancements including Google Business Profile source, Salesforce Marketing Cloud data extensions, and improved Microsoft & Google Ads sources.
- **User, admin, and audit experience upgrades.** Workspace landing page preference, Secret manager page, Trace Audit logs for admins, and improved onboarding/user navigation.
- **Performance and reliability fixes.** High CPU usage eliminated in Kubernetes workflows, BigQuery destination query timeouts, pipeline run/write collision protection, and more.

---

## New capabilities

- **Real-time multiplayer support for pipeline editing**
  - See which users are concurrently viewing or editing a pipeline.
  - Block-level live locks prevent multiple users from editing the same code block at once.
  - Inline avatars, live locking UI, and active-user banners enhance teamwork and prevent accidental overwrites.
  - Overwrite prompts and autosave guardrails protect against accidental clobbers in shared editing scenarios.

- **Release changelog dashboard**
  - Home dashboard now displays "What's new", listing release note markdown for your current Mage Pro version.
  - Release feed includes version/date tags, headings, concise highlights, and detailed feature notes.
  - Release-note content focuses strictly on product-facing, customer-visible changes.

- **Workspace landing page preference**
  - Set your preferred start screen (e.g., Pipelines, Monitoring, Runs, Triggers) for each workspace in Settings.
  - Preference is remembered per browser/device.

- **Admin-only Trace Audit page**
  - Site and workspace admins can now view a full audit log of API events.
  - Includes search, tabbed browsing, row-level filtering, CSV export, and detailed event views.

- **Secret management UI**
  - Manage secrets securely from the revamped Secrets page:
    - Resize-aware panels, stable sorting, consistent navigation, and enhanced detail view.
    - Refined copy/delete UX with accessible modal overlays.
    - Full support for Doppler secret interpolation with caching.

- **New data integration sources**
  - **Google Business Profile** source: Sync accounts, locations, performance, and more with robust error handling.
  - **Salesforce Marketing Cloud**: Dynamic Data Extension/Custom Object stream support (schema auto-resolving).
  - **Service account authentication for Google Ads**: Use service account JSON for better security.
  - **Microsoft Ads, Tiktok ads, Google Ads**: Broadened compatibility, improved sample data, schema corrections.

---

## Improvements

- **Collaboration, conflict safety, and user experience**
  - Pipeline run table and logs support "Fix with AI," surfacing actionable next-steps for failed runs and logs, right where issues occur.
  - Editor/AI "Fix" buttons and triple-quoted output highlight, plus improved comment rendering in generated code.
  - Improved onboarding navigation and user avatar selection; account settings and security streamlined.
  - Smoother pipeline creation workflow—create from scratch, template, or upload, all via modal with drag/drop, error recovery, and deduplication.

- **Reliability and operational fixes**
  - Kubernetes API high CPU usage dramatically reduced—performance patch in place.
  - BigQuery destination queries now time out reliably, preventing stuck sync jobs.
  - Trigger config parsing is robust: misconfigured YAML or missing fields no longer break pipeline loading.
  - Secret source panel and popup behavior is now pixel-perfect and theme-aware.
  - GitHub and multi-user credential hydration fixes for deployment, user/Git access, and sync isolation.

- **Data connect and pipeline sync upgrades**
  - SQL and BigQuery sources now correctly handle hyphens in dataset and table names.
  - Microsoft Ads metrics and schemas now accept string/null for missing/percent-formatted metrics, preventing sync failures to BigQuery.
  - Tiktok, Google Ads connectors upgraded for field compatibility, sample data support, and better error logging.
  - dbt integration upgraded to version 1.10.x with full dependency resolution; adapters refreshed for all key data warehouses.

- **Cloud agent/dev workflow**
  - AWS agent dev slots default to EKS; all local QA/dev work now runs on EKS slots. EC2 path removed for centralized, predictable agent slot management.
  - Fast file sync guardrails: existing slots and live slots use direct file sync only; manual archive refresh requires explicit opt-in.
  - EKS runtime and diagnostic surfaces hardened: live health checks, rollout diagnostics, and guardrails for the operational lifecycle.
  - Frontend test/generation, screenshots, and visual QA strictly use EKS; Docker/local dev stack is now reference material only.

## Reliability and fixes

- **Pipeline and workflow**
  - Pipeline blocks without UUIDs now safely ignored—no page crashes.
  - DB session safety: Pipeline triggers protected against invalid transaction states.
  - Fix cancel-all-running logic and context menu action positioning.

- **Frontend, UI, and accessibility**
  - AI sidekick output: triple-quoted comments styled as comments for readability.
  - Consistent modal, navbar, and dashboard control rendering in all themes.
  - Popup menus and delete modal contrast fixed for accessibility in light/dark themes.
  - Resizable/stacked modals and scroll-locked backgrounds for all major panel flows.
  - Improved error and warning copy, clearer tooltip labeling, more robust tab/panel hydration.

- **Infrastructure and compatibility**
  - Support for Python 3.11 and 3.12, plus conditional dependency pins for broader interpreter compatibility.
  - dbt, pandas, ClickHouse, Redshift, Snowflake, OpenTelemetry, etc. upgraded to latest supported major versions.
  - Kafka, SQLAlchemy, Spacy, and other package compatibility hardened for future-proofing.

- **Dev process, agent workflow, and release management**
  - Cloud agent process now EKS-only from creation through publish-ready and visual QA.
  - Release notes strictly filter to user-facing changes; internal/infra/developer process work is excluded from product communications.
  - Release markdown and metadata published and rendered as install-matched feed on the Home dashboard.
