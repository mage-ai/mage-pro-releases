# Workspace Foundations

A foundation release that makes workspace setup, governance, and everyday navigation feel faster and more predictable.

## Highlights

- Provision workspaces with cleaner defaults.
- Keep permission reviews fast with clearer role summaries.
- Load large project dashboards with fewer blocking requests.

## Workspace Management

New workspace defaults reduce setup time for teams onboarding additional projects. The first run now feels more guided, with fewer configuration choices required before teams can start building.

## Governance

Role summaries are easier to review before granting access. Admins can quickly distinguish viewer, editor, operator, and admin responsibilities without digging through every permission flag.

## Performance

Large dashboard views now prioritize critical content first. Navigation should feel more responsive when a project has many pipelines, schedules, users, or recent runs.

## Upgrade Notes

This sample release is safe to use for visual QA. It intentionally includes common markdown structures:

- Lists
- Tables in newer releases
- Inline `code`
- External links
- Multiple headings
