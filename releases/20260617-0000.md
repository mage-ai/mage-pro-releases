## Highlights

- **Multiplayer pipeline editor:** Real-time collaborative pipeline editing, live presence avatars, and per-block edit locks help prevent lost work from concurrent edits.
- **AI Sidekick improvements:** "Fix with AI" is more discoverable for failed runs and blocks, with streamlined UX for logs, pipeline runs, and pipeline editor integration.
- **New workspace capabilities:** Customizable landing page, improved "New Pipeline" creation, block visibility tabs, and persistent left navigation in the pipeline editor.
- **End-to-end trigger and run management upgrades:** Cancel all active pipeline runs, bulk-disable all triggers, and improved Microsoft Ads/Google Ads/BigQuery and Salesforce Marketing Cloud sources.
- **Mage Pro release notes:** Release changelogs now published and visible in the UI for every Mage Pro image.
- **Cloud agent dev: EKS-only, improved slots, live QA URLs:** Hardens AWS/EKS dev slot process, fast file transfer for remote slots, improved status reporting, and QA credentials for feature test/visual QA.
- **Security and admin tools:** Modernized secret management dashboard, Doppler secret interpolation, and an API dashboard/trace audit for workspace-wide activity.

---

## New capabilities

### Collaborative Multiplayer Pipeline Editor

- **See who’s editing:** Live avatars show which users are viewing/editing a pipeline and which block one is focused on.
- **Edit locks and conflict prevention:** Block editors become read-only if another user is editing, and a warning is displayed before overwriting a teammate’s unsaved changes.
- **Autosave and safe saves:** Autosave warns if teammates are present and would be affected. Keyboard shortcut saves (Ctrl/Cmd+S) always perform the normal save.
- **Instant presence:** Users join/leave immediately visible; blocks update in real time across tabs and browsers.

### Improved AI Sidekick and Errors

- **Fix with AI now always visible** on failed pipeline runs, blocks with errors, and directly from log error panels.
- **One-click "Fix" buttons** in pipeline runs, logs, and the pipeline editor (block header/footer) seed the AI assistant with relevant context.
- **Cleaner error detection:** Triple-quoted multiline Python errors and stack traces render as comments and are parsed reliably, showing “Fix” affordances only when actual errors are present.

### Custom Workspace Landing Page

- **Set your initial start screen** (pipelines, monitoring, runs, or triggers) from the workspace preferences menu; per-browser setting makes your workflow faster.

### Simpler Pipeline Management and Creation

- **Streamlined "New Pipeline":** Creation happens in modals with templates, file upload, and type selection all in one discoverable dropdown.
- **Random naming suggestions:** "Name" suggestions for pipelines are now shown as placeholders, not forced inputs.
- **Persistent left navigation:** The pipeline editor's left sidebar/navigation now always stays present for improved navigation and orientation.

### Block Visibility and Editor Enhancements

- **Block visibility tabs are back:** Quickly switch between "All files" and "Current blocks" in the pipeline editor left panel, and show/hide block editors as needed.
- **Resizable panels:** Secrets detail panels, audit logs, and file browser sidebars support intuitive resizing.

### Triggers, Schedules, and Run Controls

- **Cancel all running pipeline runs:** Bulk cancel all "in progress" runs at once from the global runs page.
- **Bulk-disable triggers:** Disable all active triggers for a project with a single action.
- **Cleaner one-off trigger presentation:** Terminal "@once" triggers show only an actionable control, not irrelevant toggles.

### Secret Management & Integration

- **Secrets dashboard:** Modernized UI with sorting, scrolling, better create secret panel, and clear action affordances.
- **Doppler secret interpolation:** Use Doppler secrets (auto-refreshed with low TTL) in workspace configs and variables, enabling seamless secret rotation without server restart.

### Data Sources, Integrations, and Reliability

- **Expanded data sources:**
    - **Google Business Profile**: Adds account, locations, search keywords, reviews, and more.
    - **Salesforce Marketing Cloud**: Dynamic custom object/data extension stream support.
    - **Microsoft Ads/Google Ads/BigQuery**: Improved handling of metric types, errors, sample data, hyphenated table names, field conflicts, and service account auth.
- **DBT 1.10.x support:** Upgrades DBT across the stack for the latest features and compatibility.
- **Python 3.11 / 3.12 support:** Run Mage Pro on modern Python versions.

### Mage Pro Release Notes Feed

- **Changelog publishing:** Each Mage Pro Docker image's release notes are now published and rendered on the workspace Home page, always in sync with your running version.

### Improved Admin Tools

- **Trace Audit dashboard:** Admins can view, search, and filter a workspace's API events/changes with a dedicated dashboard and full detail panel.

## Improvements

- Reliable process queue size reloading without restart.
- BigQuery destination jobs now respect query API timeouts and fail gracefully if queries hang.
- Cross-session Git token scoping: prevents leaking user-specific tokens across accounts.
- Enhanced server stability w.r.t. Kubernetes client high CPU usage and memory calculation for containerized environments.
- Secret management: fixed panel resizing, delete popup contrast, and empty state bugs.
- Faster, more reliable frontend and backend CI: sharded pytest, frontend CI separation, and improved caching.
- UI: Compact dashboard resizes, friendlier trace audit panel, always-visible left navigation, and improved dark/light mode contrast everywhere.
- Triggers: made scheduler config fault-tolerant and retroactively fixed cron schedule parsing crash in status-sorted triggers view.

---

## Reliability and fixes

- Block logs on S3 and GCS are now surfaced and accessible in the Logs panel.
- Fixes for BigQuery table/dataset naming, metric schema alignment, and sample data handling.
- OAuth sign-in fixes: no default role granted; improved redirect support for base paths and nested workspaces.
- Fixed pipeline graph dependency mapping bug when a block was missing a `uuid`.
- Numerous Python/pandas warnings, SQLAlchemy queries, and test suite speed/stability improvements.
- Download and file operations now allow editor-role users to upload/download files outside the project folder with clear permission checks in place.
- Prevented pipeline run stats miscounting workspace/run ownership.
- Bulk trigger/schedule actions keep project YAML and DB state consistent, rolling back changes on failure.
- UI fixes: block configure modal rendering, AI sidekick modal focus state, command bar/AI panel display bugs, consistent coloring for multiline/triple-quoted Python comments, missing nav toggle and resizable nav, menu disables, button states, and tooltip copy.
- Ensured Microsoft Ads/Facebook Ads/Google Ads connector error messages surface real API responses for easier debugging.
